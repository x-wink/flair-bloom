## 二、第三方驱动组件

本软件提供多种输入模式以兼容不同游戏，其中部分模式依赖第三方内核驱动。您需了解并同意以下事项：

### 2.1 Interception 驱动模式

集成 **Interception**（由 Francisco Lopes 开发，LGPL-3.0 许可证）内核驱动组件，用于在 Raw Input API 类游戏中提供驱动级按键模拟，对应输入模式「游戏模式」。

1. **安装须知**：启用「游戏模式」需要安装 Interception 内核驱动，安装过程需要管理员权限，安装完成后需要重启计算机。
2. **系统风险**：该驱动作为内核级组件运行，在极少数情况下可能导致系统输入异常（如登录界面键盘无响应）。如遇此情况，可通过安全模式启动系统并卸载驱动恢复。
3. **兼容性风险**：部分游戏的反作弊系统（如 EA Anti-Cheat）可能检测到 Interception 驱动并拒绝启动游戏。此为该驱动的已知限制，非本软件缺陷。

### 2.2 DD 驱动模式

集成 **DD 虚拟键鼠 SDK**（作者 ddxoft，[github.com/ddxoft/master](https://github.com/ddxoft/master)）相关运行时组件，用于在「DD驱动」模式下兼容更严格的 Raw Input 类游戏。

1. **运行权限**：「DD驱动」要求宿主进程以管理员身份运行。切换至此模式时，本软件会请求管理员授权并自动重启自身。
2. **运行方式**：该模式运行时加载随应用打包的 `dd63330.dll`，并通过 DD SDK 自检确认驱动通道是否就绪。若资源缺失、校验异常、权限不足或驱动通道未就绪，该模式可能无法启用或回退到「通用模式」。
3. **输入支持与限制**：当前「DD驱动」模式支持键盘、鼠标、滚轮和侧键注入，但实际兼容性仍取决于系统环境、目标程序和反作弊策略。本软件不保证该模式在所有游戏或设备上可用。
4. **系统风险**：DD 驱动属于系统级输入注入方案，与 Interception 同样存在与反作弊机制冲突、被目标程序拒绝、或在罕见硬件/系统环境上引发输入异常的可能。
5. **来源声明**：DD 相关运行时组件（包括 DD DLL、DDHID DLL、SYS、INF、CAT 及 ddc.exe）均按第三方发布的二进制原样集成，本软件未对其进行重新签名。详情参见 `THIRD_PARTY.md`。

### 2.3 DDHID 模式暂停使用

经测试，原「DDHID」驱动安装链路存在不稳定风险。当前版本已暂停开放 DDHID 模式，用户主动切换 DDHID 会被阻止；加载旧配置或旧状态命中 DDHID 时，本软件可能自动回退到「通用模式」并提示用户处理。

1. **暂停范围**：暂停的是 DDHID 模式入口及其安装链路，不影响「通用模式」「游戏模式」和「DD驱动」模式的正常使用。
2. **已安装驱动处理**：如果您的系统中已经安装 DDHID 驱动，建议在本软件「诊断修复」中执行卸载。卸载后如提示待重启，应按提示重启计算机。
3. **残留风险**：DDHID 驱动残留可能导致驱动重装失败、输入异常或与系统安全策略/反作弊组件冲突。本软件提供诊断与修复工具，但不保证能够在所有系统环境中自动修复所有残留问题。

### 2.4 通用条款

1. **可选性**：所有非通用输入模式均为可选功能，默认使用「通用模式」（SendInput），无需安装任何驱动即可运行本软件全部基础功能。
2. **卸载方式**：
   - Interception：在本软件「诊断修复」中执行卸载操作，或以管理员身份运行 `install-interception.exe /uninstall` 后重启计算机。
   - DD驱动：卸载本软件会移除随应用打包的 DD DLL；如系统中存在其他来源安装的 DD 驱动或服务，请按其来源组件说明处理。
   - DDHID：在本软件「诊断修复」中执行卸载，或以管理员身份运行驱动目录下的 `ddc.exe -u`。
