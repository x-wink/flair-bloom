## 二、第三方驱动组件

本软件提供多种输入模式以兼容不同游戏，其中部分模式依赖第三方内核驱动。您需了解并同意以下事项：

### 2.1 Interception 驱动模式

集成 **Interception**（由 Francisco Lopes 开发，LGPL-3.0 许可证）内核驱动组件，用于在 Raw Input API 类游戏中提供驱动级按键模拟，对应输入模式「游戏模式」。

1. **安装须知**：启用「游戏模式」需要安装 Interception 内核驱动，安装过程需要管理员权限，安装完成后需要重启计算机。
2. **系统风险**：该驱动作为内核级组件运行，在极少数情况下可能导致系统输入异常（如登录界面键盘无响应）。如遇此情况，可通过安全模式启动系统并卸载驱动恢复。
3. **兼容性风险**：部分游戏的反作弊系统（如 EA Anti-Cheat）可能检测到 Interception 驱动并拒绝启动游戏。此为该驱动的已知限制，非本软件缺陷。

### 2.2 DDHID 模式（DD 驱动）

集成 **DD 虚拟键鼠 SDK**（作者 ddxoft，[github.com/ddxoft/master](https://github.com/ddxoft/master)）的 HID-Class 驱动变体，用于兼容更严格的 Raw Input 类游戏。

1. **运行权限**：「DDHID」底层调用 `DeviceIoControl`，要求宿主进程以管理员身份运行。切换至此模式时本软件会请求管理员授权并自动重启自身。
2. **安装方式**：作为 PnP HID-Class 驱动安装，使用内置的 `ddc.exe` 工具完成安装/卸载，无需重启计算机；驱动包已通过微软 WHQL 签名（HVCI 兼容）。
3. **使用限制**：DD 驱动无法对自身注入的按键进行标识，本软件在按压连发模式下通过应用层注入事件队列识别自循环，但「切换连发」模式因开关逻辑无法过滤注入事件，因此切换连发模式下**「目标键」不能与「启动热键」或「停止热键」相同**。本软件会在切换模式与编辑规则时强制校验。
4. **系统风险**：此驱动为内核级组件，与 Interception 同属系统级注入方案，存在与反作弊机制冲突或在罕见硬件上引发输入异常的可能。
5. **来源声明**：DD 驱动的所有运行时组件（DLL、SYS、INF、CAT 及 ddc.exe）均按原作者发布的二进制原样集成，本软件未对其进行任何修改或重新签名。详情参见 `THIRD_PARTY.md`。

### 2.3 通用条款

1. **可选性**：所有驱动模式均为可选功能，默认使用「通用模式」（SendInput），无需安装任何驱动即可运行本软件全部基础功能。
2. **卸载方式**：
   - Interception：在本软件设置中执行卸载操作，或以管理员身份运行 `install-interception.exe /uninstall` 后重启计算机。
   - DDHID：在本软件设置中执行卸载，或以管理员身份运行驱动目录下的 `ddc.exe -u`。
