## [0.2.1] - 2026-05-29

### 新功能

- **外部配置导入**：支持从丐帮高手（`config.json`）一键导入配置。点击配置文件菜单中的「导入外部配置…」，自动扫描桌面/下载目录；识别连发规则、连发间隔、全局开关热键、停止热键、面板热键后预览确认，导入为独立配置文件。手动粘贴路径同样支持
- **全局热键**：规则列表上方新增热键设置区，支持绑定「全局开关开启键 / 停止键（可选，默认同开启键作切换）」与「面板显隐键」。热键属于配置文件，随配置切换；应用启动时立即生效，无需等待前端加载
- **切换连发语音播报**：全局开关启用时播报「我准备好库库按了」，禁用时播报「我累了歇会」，使用系统中文 TTS，仅在用户主动切换时触发

### 问题修复

- 关闭全局开关时（热键触发或手动点击）立即中止所有正在运行的连发循环，防止按键卡住
- 删除当前配置后，前端热键显示正确同步为回退配置的热键
- 修改默认配置的热键时，与修改规则行为一致，自动 fork 出新配置

### 行为变更

- 按键捕获控件：右键点击已设置的按键可清除绑定；进入捕获状态 5 秒无操作自动退出；Esc 现在可正常绑定为热键
- 热键设置从底栏移至规则列表上方，属于配置级别设置；底栏仅保留全局设置（输入模式、全局开关）
- 图标更新，覆盖 Windows 安装包、macOS .app、系统托盘及所有平台尺寸变体

### 升级方式

覆盖安装新版安装包即可，旧版 `.qzh` 配置完全兼容（新增热键字段向后兼容，自动补默认值）。

## [0.2.0] - 2026-05-28

### 新功能

- **键盘全键位录入**：按键录入器（启动热键 / 连发键 / 停止热键）支持键盘所有刻字键，包含 F13–F24、小键盘 0–9 与 `* / + - .`、OEM 标点（`; = , - . / \` `[ ] '`）、Insert / Delete / Home / End / PageUp / PageDown 等编辑键，以及 CapsLock / NumLock / ScrollLock / Pause / 菜单键
- **鼠标连发**：连发的触发键与目标键均可设置为鼠标左键 / 右键 / 中键 / 侧键 1 / 侧键 2，鼠标键既可作为触发热键也可作为连发输出。在按键录入处于「按下按键…」状态时，按下任意鼠标键即录入
- **三档输入模式覆盖鼠标连发**：通用模式（SendInput）、游戏模式（Interception）、究极HID 三档对鼠标连发同时生效，覆盖不同反作弊场景

### 行为变更

- 配置文件 schema 升至 v2：所有按键字段从原始虚拟键码（u32）改为带类型标记的 `KeyId`（`{kind:"keyboard",code:0x51}` 或 `{kind:"mouse",code:"left"}`）。打开旧版本配置时会自动迁移，无需手动操作
- 究极HID 模式不支持鼠标侧键（X1/X2）作为「连发目标键」，DD SDK 协议所限。如需在究极HID 下连发侧键，请改用左/右/中键或键盘键作为目标；侧键作为触发键不受影响
- 切换输入模式时引擎会同时清理键盘与鼠标 hook，避免残留监听

### 升级方式

覆盖安装新版安装包即可。旧版 `.qzh` 配置首次打开会自动升级到 v2，并按当前应用版本重写。

### 已知问题

- WebView2 仍可能在某些环境下吞掉键盘 `e.code === 'NumpadEnter'`，已显式映射回 VK_RETURN 0x0D 与主键盘 Enter 行为一致

## [0.1.11] - 2026-05-27

### 新功能

- **环境诊断「安装前置检查」**：在原有驱动残留检测之外，新增检查内核隔离 (HVCI) / Smart App Control / 系统待重启 / Windows Defender 白名单等可能阻断驱动首次安装的系统策略，并给出对应处理引导

### 问题修复

- **驱动「待重启」状态显示与切换**：卸载 DD-HID 驱动后未重启电脑时，关于弹窗错误地显示为「未安装」；现统一以驱动 `.sys` 文件 + 服务键交叉判定，正确区分「已安装 / 待重启 / 未安装」三态。Interception 驱动同样适用
- **半卸载状态下切换输入模式不再触发重装**：检测到驱动处于「待重启」状态时，切换到该模式将提示「请重启电脑」而不再调起安装流程，避免 PnP 拒绝重新注册导致的 install error
- **关于弹窗的卸载按钮**：仅在驱动「已安装」时显示，「待重启」状态下隐藏，防止重复触发卸载

### 行为变更

- DD-HID 驱动安装判定从「sys 文件落盘」升级为「sys 文件 + 服务键同时就位」；半卸载残留场景将给出「请重启电脑后再安装」的明确提示

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容。

## [0.1.10] - 2026-05-27

### 新功能

- **关于弹窗整合诊断信息**：合并原「关于」与状态面板，集中展示系统版本 / WebView2 / 主机架构 / 用户区域 / 安装路径，以及驱动安装、开机自启、资源完整性等排查项；后端新增 `app-status-changed` 事件主动推送，弹窗内容随状态实时刷新
- **应用目录快捷入口**：弹窗内可一键打开安装目录、用户数据目录、日志目录（白名单受控），原顶部「查看日志」入口已合并至此

### 问题修复

- **HID 驱动卸载**：`ddc.exe` 退出码在部分系统下不可靠，安装/卸载改为以驱动 `.sys` 文件存在与否为权威信号
- **TrustedInstaller ACL 阻挡卸载**：当 HID 驱动文件被 TrustedInstaller 占有导致常规卸载失败时，现自动通过提权 PowerShell 执行 `takeown` + `icacls` 强删；仍无法删除时写入 `PendingFileRenameOperations`，提示用户重启电脑后自动清理
- **切换输入模式残留句柄**：从 DD-HID / Interception 切回通用模式时主动释放后端句柄，避免驱动文件被占用

### 行为变更

- `uninstall_dd_hid_driver` 命令返回结构体（区分「卸载完成」/「需重启」/「失败」），前端可据此给出不同提示

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容。

## [0.1.5] - 2026-05-26

### 新功能

- **三档输入模式**：将原「驱动增强 开/关」开关重构为可选模式列表，新增 **究极HID**（基于 ddxoft DD 虚拟键鼠 SDK 的 HID-Class 驱动），覆盖 Interception 仍无效的游戏（如**剑网三**）
  - 通用模式（SendInput，默认）：最稳定，但很多游戏不响应
  - 游戏模式（Interception）：兼容多数游戏，需安装内核驱动
  - 究极HID（DD-HID）：极致兼容，剑网三等 Raw Input 严格过滤的游戏需用此模式；基于 PnP HID-Class 驱动，使用内置 `ddc.exe` 安装/卸载，无需重启电脑，HVCI 友好（驱动包通过 WHQL 签名），需以管理员身份运行
- **管理员授权**：究极HID 模式底层调用 `DeviceIoControl` 需要管理员权限。切换至此模式时，本软件会通过 UAC 弹窗请求授权并自动重启自身，无需用户手动以管理员方式启动
- **激活态视觉反馈**：连发规则正在执行时（按压模式触发键被按住、切换模式已开启），规则卡片显示呼吸脉冲动画，便于一眼区分「已启用」与「正在连发」

### 问题修复

- 修复 dev 构建下提权重启或直接执行 `target/debug/flair-bloom.exe` 时报「找不到 interception.dll」无法启动的问题：构建脚本现在会把 `interception-sys` 解压出的 DLL 复制到 EXE 同级目录

### 行为变更

- 用户协议升级至 v1.2，新增 DD 第三方驱动声明、运行权限说明与卸载方式，升级后需重新确认协议
- 究极HID 模式下「切换连发」的目标键不能与启动/停止热键相同（DD 驱动无法标识自身注入的按键），切换模式或保存配置时会拦截并提示；「按压连发」不受此限制
- 按压连发的「按压键」改为高级设置折叠项，默认与「连发按键」相同，符合「等技能 CD 好就连按 Q」的常见用法；如需异键触发可展开高级设置单独配置
- 新装应用的默认配置文件名由 `默认配置.qzh` 改为 `defults.qzh`（已升级用户保持原文件不受影响）

### 已知问题

- DD-HID 驱动作为内核级组件，可能与部分游戏的反作弊机制冲突；若启用后游戏拒绝启动，请切换回「通用模式」或「游戏模式」

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容。如需启用究极HID 模式，在面板底部点击当前模式名称（默认为「通用模式」）切换；首次切换会触发 UAC 提权并自动安装 HID 驱动，无需重启电脑。

## [0.1.1] - 2026-05-25

- **扩展键连发无效**：方向键、Delete、Home、End 等扩展键因扫描码前缀处理错误无法正常连发，本次一并修正
- **停止连发时目标键偶发卡住**：重写了线程退出与按键释放的协调逻辑，确保键位正确抬起
- **切换规则时旧线程短暂残留**：保证新旧线程不再重叠运行

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容，无需重新设置规则。
