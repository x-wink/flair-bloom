所有版本的更新记录。发版前在 `[Unreleased]` 节填写内容，运行 `pnpm bump-version` 后自动转为对应版本节。

## [Unreleased]

## [0.1.6] - 2026-05-26

### 问题修复

- 修复 NSIS 安装包（setup.exe）在未预装 Interception 驱动的机器上启动时报「找不到 interception.dll」无法运行的问题：构建脚本现在会把 DLL 同步到 `resources/`，安装包通过新增的 NSIS hook 在 PostInstall 阶段把它移动到 EXE 同级目录，卸载时一并清理

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容。

## [0.1.5] - 2026-05-26

### 新功能

- **三档输入模式**：将原「驱动增强 开/关」开关重构为可选模式列表，新增 **究极HID**（基于 ddxoft DD 虚拟键鼠 SDK 的 HID-Class 驱动），覆盖 Interception 兼容性不足的少数 Raw Input 类游戏（如剑网三）
  - 通用模式（SendInput，默认）：最稳定，但很多游戏不响应
  - 游戏模式（Interception）：兼容多数游戏，需安装内核驱动
  - 究极HID（DD-HID）：极致兼容，基于 PnP HID-Class 驱动，使用内置 `ddc.exe` 安装/卸载，无需重启电脑，HVCI 友好（驱动包通过 WHQL 签名），需以管理员身份运行
- **管理员授权**：究极HID 模式底层调用 `DeviceIoControl` 需要管理员权限。切换至此模式时，本软件会通过 UAC 弹窗请求授权并自动重启自身，无需用户手动以管理员方式启动
- **激活态视觉反馈**：连发规则正在执行时（按压模式触发键被按住、切换模式已开启），规则卡片显示呼吸脉冲动画，便于一眼区分「已启用」与「正在连发」

### 问题修复

- 修复 dev 构建下提权重启或直接执行 `target/debug/flair-bloom.exe` 时报「找不到 interception.dll」无法启动的问题：构建脚本现在会把 `interception-sys` 解压出的 DLL 复制到 EXE 同级目录

### 行为变更

- 用户协议升级至 v1.2，新增 DD 第三方驱动声明、运行权限说明与卸载方式，升级后需重新确认协议
- 究极HID 模式下「切换连发」的目标键不能与启动/停止热键相同（DD 驱动无法标识自身注入的按键），切换模式或保存配置时会拦截并提示；「按压连发」不受此限制
- 按压连发的「按压键」改为高级设置折叠项，默认与「连发按键」相同，符合「等技能 CD 好就连按 Q」的常见用法；如需异键触发可展开高级设置单独配置
- 新装应用的默认配置文件名由 `默认配置.qzh` 改为 `defults.qzh`（已升级用户保持原文件不受影响）

### 已知问题

- DD-HID 驱动作为内核级组件，可能与部分游戏的反作弊机制冲突；若启用后游戏拒绝启动，请切换回「通用模式」或「游戏模式」

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容。如需启用究极HID 模式，在面板底部点击当前模式名称（默认为「通用模式」）切换；首次切换会触发 UAC 提权并自动安装 HID 驱动，无需重启电脑。

## [0.1.4] - 2026-05-25

### 问题修复

- 修复驱动模式下「触发键 = 目标键」时切换连发（Toggle）会被自身注入的按键立即关闭的问题：Interception 注入事件现在会写入与 SendInput 相同的过滤标记，键盘钩子可正确识别并跳过

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容，无需重新设置规则。

## [0.1.3] - 2026-05-25

### 新功能

- **驱动增强模式**：新增 Interception 内核驱动支持，解决剑网三等使用 Raw Input API 的游戏中按键模拟无效的问题。在面板底部点击「驱动增强」开关即可启用，首次使用需安装驱动并重启电脑
- **双模式架构**：标准模式（SendInput）与驱动模式可自由切换，默认标准模式，兼容性最佳；驱动模式兼容更多游戏

### 行为变更

- 用户协议更新至 v1.1，新增第三方驱动组件声明与风险提示，升级后需重新确认协议

### 升级方式

覆盖安装新版安装包即可。如需使用驱动增强功能，点击面板底部「驱动增强」开关，按提示安装驱动并重启电脑。

### 已知问题

- Interception 驱动在极少数 Windows 11 设备上可能导致登录界面键盘无响应，可通过安全模式卸载恢复
- 部分游戏反作弊（如 EA Anti-Cheat）会检测 Interception 驱动并阻止游戏启动

## [0.1.2] - 2026-05-25

本次重点修复了自动更新流程，并完善了更新体验。

### 新功能

- **自动更新**：新版本发布后，应用将在后台静默下载安装包，下次启动时自动完成安装，无需手动操作
- **更新公告**：新增「更新公告」弹窗，展示版本说明；下拉菜单可随时回看，有待安装更新时以红点提示

### 问题修复

- 修复版本号显示：关于页面版本号现从系统读取，不再与实际版本不符
- 修复扩展键（方向键、Insert、Delete 等）注入时编码错误导致按键无效的问题
- 修复按住时长精度问题，`hold_ms` / `rest_ms` 控制更准确
- 修复引擎多次重建可能挂载双重键盘钩子的问题
- 修复退出时极端情况下可能遗漏 key_up 的竞态问题

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容，无需重新设置规则。

## [0.1.1] - 2026-05-25

本次为紧急修复版本，解决了 v0.1.0 中连发功能在游戏内无效的根本性问题。

### 问题修复

- **连发在游戏中完全无效**（v0.1.0 已知问题）：改为硬件扫描码注入，游戏内连发恢复正常，强烈建议所有 v0.1.0 用户立即升级
- **扩展键连发无效**：方向键、Delete、Home、End 等扩展键因扫描码前缀处理错误无法正常连发，本次一并修正
- **停止连发时目标键偶发卡住**：重写了线程退出与按键释放的协调逻辑，确保键位正确抬起
- **切换规则时旧线程短暂残留**：保证新旧线程不再重叠运行

### 升级方式

覆盖安装新版安装包即可，配置文件完全兼容，无需重新设置规则。

## [0.1.0] - 2026-05-24

首个公开内测版（MVP），功能基本可用。

### 新功能

- **连发引擎**：按压连发（Hold）、切换连发（Toggle）、多规则并行、全局开关
- **按键录入**：点击输入框直接按下目标键完成绑定
- **无边框透明窗口**：可在桌面任意拖动
- **系统托盘常驻**：关闭窗口后程序继续在后台运行，连发保持有效
- **开机自启**：支持随系统启动，可在托盘菜单中关闭
- **自动更新**：启动时静默检查新版本（v0.1.2 修复实际可用）
- **配置加密存储**：配置文件以 .qzh 格式存储，采用 AES-256-GCM 加密

### 已知问题

- 游戏内连发无效（已在 v0.1.1 修复）
