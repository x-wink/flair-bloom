# 更新日志

所有版本的更新记录。发版前在 `[Unreleased]` 节填写内容，运行 `pnpm bump-version` 后自动转为对应版本节。

## [Unreleased]

## [0.3.0] - 2026-06-29

### 新功能

- **常驻置顶浮窗**：最小化（或关闭时选「切换到悬浮窗」）后，桌面右下角留下一个极简悬浮胶囊——常驻置顶、可拖动、跟随主题，实时显示当前正在连发的按键，内置「启动 / 暂停」全局开关，点右侧按钮即可展开回主面板。位置会被记住，下次启动自动复位（坐标越界时回到默认位置）。
- **横版键鼠图布局**：标题栏新增布局切换，可从竖版规则列表切到「横版键鼠图」——在键盘 / 鼠标示意图上直接点键设连发（点一下设为切换连发、再点改按压、再点取消），一眼看清哪些键在用。
- **主题换肤 + 暗色模式**：设置新增外观一节，内置 21 个剑三门派配色预设色板，并支持亮 / 暗 / 跟随系统三种明暗模式；主题色在全应用（含浮窗）统一生效。全局开启时主面板与浮窗背景改为右下径向渐变 + 缩放呼吸动效。
- **声音播报每项独立开关**：全局开始 / 结束、Toggle 开始 / 结束四个播报时机现在可分别开关（开关滑块样式），试听按钮内嵌到文本框中。

### 问题修复

- **修复以管理员重启「只退出不重启」**：切换游戏模式 / DD驱动需要提权时，重启不再只退不起；新增提权重启入口与「等前任退出释放单实例锁再启动」握手，避免重复实例冲突。
- **修复浮窗聚焦时热键被吞**：悬浮窗获得焦点时全局键盘钩子失效会导致热键无响应，现与主面板共用键盘事件中继，聚焦任一窗口热键都正常。
- **提升后端切换卡键与停止可靠性**：切换输入后端 / 停止 / 退出时更彻底地复位按键状态，减少卡键与「收不住」。
- **修正默认配置名拼写**：历史拼写 `defults` 加载时自动迁移为 `defaults`。

### 行为变更

- **关闭 / 最小化改为「切换到悬浮窗」**：关闭确认与设置里原「最小化到托盘」改为「切换到悬浮窗」，关闭时默认收起为右下角悬浮窗、连发继续运行。
- **DD驱动与横版键鼠图互斥**：DD驱动（含 DDHID）模式下不支持横版键鼠图——横版下禁止切到 DD驱动、DD驱动下禁止切到横版，界面会相应置灰提示。
- **声音播报新增每项独立开关**：其中 **Toggle 开始 / 结束播报默认关闭**——若你此前依赖切换连发时的语音提示，请在「设置 → 声音」中手动开启。

### 升级方式

- 覆盖安装新版安装包即可，配置文件完全兼容、自动迁移。

## [0.2.9] - 2026-06-24

### 行为变更

- **现在主推「游戏模式」**：输入模式列表重新排序与标注——「游戏模式」（Interception 驱动）为**推荐**首选，「DD驱动」降为**备用**（仅在游戏模式不可用时使用），「DDHID」**已禁用**、不再出现在可选列表。原因：DD 系列驱动在部分场景下可能无法正确停止连发、甚至自行停掉连发，稳定性不足。优先使用游戏模式体验最稳。
- **多条规则同时连发时自动限速**：当多条连发规则一起工作时，软件会自动平摊总的按键速度——每条规则的最快速度 = 基础速度 ÷ 同时运行的规则数，使总速率保持稳定、与规则数量无关。这样可避免多条规则叠加把按键发得太快，导致**松手后还在狂按、停不下来**的问题。只开一条规则时不受影响。
- **连发间隔最小值恢复为 10ms**：上一版开放到 1ms「极速档」，但实测过快会冲破系统输入管线上限、造成「停不住」。本版本将有效最小间隔恢复为 **10ms**（约每秒 100 次）；加载旧配置时，低于 10ms 的间隔会被自动调整为 10ms。
- **全局热键只支持键盘按键、且不能重复**：全局开启 / 停止 / 面板显隐三个热键现在只能绑定键盘按键（不再接受鼠标按键），并且三者不能设为同一个键，避免按键被某个热键抢先处理、其余功能失效导致行为混乱。

### 问题修复

- **根治多规则连发「收不住」**：修复多条规则同时连发时，松开触发键后仍持续狂按、停不下来的问题（见上「自动限速」）。
- **提升停止可靠性**：停止 / 暂停 / 切换配置 / 退出时更彻底地复位内部按键状态，清理可能残留的「幻影按键」，确保紧急停止键 100% 生效。
- **连发线程崩溃兜底**：连发后台线程一旦异常退出，会按账本自动补发按键释放，最大限度避免按键卡死。

### 升级方式

- 覆盖安装新版安装包即可，配置文件完全兼容；旧配置中低于 10ms 的连发间隔会在加载时自动调整为 10ms。

## [0.2.8] - 2026-06-17

### 问题修复

- **连发引擎稳定性强化**：连发调度改为单线程高精度定时，停止 / 切换配置 / 退出时更可靠地释放按键；新增引擎线程崩溃兜底，异常时按账本补发按键释放，降低极端情况下的卡键风险

### 行为变更

- 用户协议升级至 v1.3，补充「DD驱动」模式、DDHID 暂停使用、驱动残留与卸载说明，升级后需重新确认协议
- 连发间隔最小值由 10ms 放宽至 1ms（极速档），过低可能被系统或游戏限流，请谨慎使用

### 升级方式

## [0.2.0] - 2026-05-28

### 新功能

- **键盘全键位录入**：按键录入器（启动热键 / 连发键 / 停止热键）支持键盘所有刻字键，包含 F13–F24、小键盘 0–9 与 `* / + - .`、OEM 标点（`; = , - . / \` `[ ] '`）、Insert / Delete / Home / End / PageUp / PageDown 等编辑键，以及 CapsLock / NumLock / ScrollLock / Pause / 菜单键
- **鼠标连发**：连发的触发键与目标键均可设置为鼠标左键 / 右键 / 中键 / 侧键 1 / 侧键 2，鼠标键既可作为触发热键也可作为连发输出。在按键录入处于「按下按键…」状态时，按下任意鼠标键即录入
- **三档输入模式覆盖鼠标连发**：通用模式（SendInput）、游戏模式（Interception）、DDHID 三档对鼠标连发同时生效，覆盖不同反作弊场景

### 行为变更

- 配置文件 schema 升至 v2：所有按键字段从原始虚拟键码（u32）改为带类型标记的 `KeyId`（`{kind:"keyboard",code:0x51}` 或 `{kind:"mouse",code:"left"}`）。打开旧版本配置时会自动迁移，无需手动操作
- DDHID 模式不支持鼠标侧键（X1/X2）作为「连发目标键」，DD SDK 协议所限。如需在DDHID 下连发侧键，请改用左/右/中键或键盘键作为目标；侧键作为触发键不受影响
- 切换输入模式时引擎会同时清理键盘与鼠标 hook，避免残留监听

### 升级方式

覆盖安装新版安装包即可。旧版 `.qzh` 配置首次打开会自动升级到 v2，并按当前应用版本重写。

### 已知问题

- WebView2 仍可能在某些环境下吞掉键盘 `e.code === 'NumpadEnter'`，已显式映射回 VK_RETURN 0x0D 与主键盘 Enter 行为一致

## [0.1.11] - 2026-05-27

### 新功能

- **关于弹窗的卸载按钮**：仅在驱动「已安装」时显示，「待重启」状态下隐藏，防止重复触发卸载
