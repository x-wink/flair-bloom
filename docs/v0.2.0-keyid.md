# 扩展按键支持到鼠标 + 键盘全键位（v0.2.0）

## 背景

当前实现只覆盖少量按键：

- 前端 `KeyCapture` 白名单约 60 个 VK（A-Z / 0-9 / F1-F12 / Space / Enter / Esc / Backspace / Tab / 4 个方向键），不在表内的 `e.code` 被静默丢弃；鼠标完全没有录入入口。
- 三套注入通道（SendInput / DD-HID / Interception）的 send 方法签名均为 `send_key(vk, is_up)`，只构造键盘事件结构。`engine/dd_common.rs:6` 注释明说「鼠标功能暂不使用」，`engine/burst.rs` 只装了 `WH_KEYBOARD_LL`。
- 配置 schema `BurstRule.trigger_key/target_key/stop_key` 与 `Hotkeys.global_toggle` 都是 `u32 VK`，无法表达「键盘 VK 还是鼠标按钮」。
- License 已预留 `MOUSE_BURST = 1 << 1`（`packages/crypto/src/license.rs:16`），目前 0 引用。

目标：连发引擎能录入和注入「键盘所有刻字键 + 鼠标 5 键（左 / 右 / 中 / X1 / X2）」，作为 v0.2.0 一次性发布。本版本暂不做 license gating（功能对所有用户开放），等兑换码体系完善后再补。

## 设计：统一按键标识

引入 tagged 结构替换所有 `u32` 按键字段：

```rust
// packages/qzh-format/src/key_id.rs（新文件）
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(tag = "kind", content = "code", rename_all = "snake_case")]
pub enum KeyId {
    Keyboard(u32), // Win32 VK
    Mouse(MouseButton),
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum MouseButton { Left, Right, Middle, X1, X2 }
```

JSON 形态：`{"kind":"keyboard","code":81}` / `{"kind":"mouse","code":"left"}`，前后端共享同一编码。旧 schema v1 的 `u32` 全部按 `Keyboard(u32)` 迁移；`global_toggle` 的 `null` / 缺省保持原语义。

## 关键改动点

### 1. 数据模型 + 迁移（无运行时副作用，先合）

| 文件 | 改动 |
|---|---|
| `packages/qzh-format/src/key_id.rs` | 新增 `KeyId` / `MouseButton` 类型 |
| `packages/qzh-format/src/lib.rs` | re-export `key_id` |
| `packages/qzh-format/src/profile.rs` | `BurstRule.trigger_key/target_key: KeyId`、`stop_key: Option<KeyId>`、`Hotkeys.global_toggle: Option<KeyId>`；`CURRENT_SCHEMA_VERSION = 2`；`validate_for_mode` 加规则：DD-HID 模式拒绝 `target_key = Mouse(X1\|X2)`（DD SDK 不支持） |
| `packages/qzh-format/src/migrate.rs` | 实现 v1→v2：遍历 `rules[].trigger_key/target_key/stop_key` 与 `hotkeys.global_toggle`，把 `u32` 包装成 `{"kind":"keyboard","code":...}` |
| `packages/qzh-format/src/profile_tests.rs` | 加测试：序列化往返、新 validate 拒绝 DD-HID + Mouse X1/X2 |

`packages/migrate/src/lib.rs::run_migrations` 不需要动，已经是泛型 runner。

### 2. 注入侧三通道（核心工作量）

**`apps/main/src-tauri/src/engine/input.rs`**

- `key_down(KeyId)` / `key_up(KeyId)`：把 `vk: u32` 参数换成 `KeyId`，按 `kind` 分流。
- `send_via_sendinput` 拆成 `send_kbd_via_sendinput(vk, flags)`（保持原逻辑）+ `send_mouse_via_sendinput(button, is_up)`（新增）。
- 鼠标分支构造 `INPUT_MOUSE` + `MOUSEINPUT { dwFlags, mouseData, dwExtraInfo: SIM_MARKER }`：
  - Left → `MOUSEEVENTF_LEFTDOWN/UP`
  - Right → `MOUSEEVENTF_RIGHTDOWN/UP`
  - Middle → `MOUSEEVENTF_MIDDLEDOWN/UP`
  - X1/X2 → `MOUSEEVENTF_XDOWN/UP` + `mouseData = XBUTTON1/XBUTTON2`
- `PENDING_INJECTIONS` 的键由 `(u32, bool)` → `(KeyId, bool)`，`record_injection` / `try_consume_injection` 签名同步。
- `SIM_MARKER` 在 `MOUSEINPUT.dwExtraInfo` 同样写入，鼠标 LL 钩子据此过滤自身注入。

**`apps/main/src-tauri/src/engine/dd_common.rs` + `ddhid.rs`**

- `DdFfi` 增加 `dd_btn` 函数指针的可调用形态（目前只用作自检 `DD_btn(0)`），并新增 `send_mouse_button(MouseButton, is_up)`：
  - DD SDK 的 `DD_btn(flag)` 文档值域：`1`=左下 / `2`=左上 / `4`=右下 / `8`=右上 / `16`=中下 / `32`=中上
- DD-HID 不支持 X1/X2：`send_mouse_button` 对 X1/X2 直接 `warn!` 并 fall-through 到 SendInput 路径（与 `DD_todc` 返回 0 时的策略对称）。schema validate 已在 DD-HID 模式拒绝该组合，正常路径不会到达。
- `DdHidBackend` 新增 `send_mouse(MouseButton, is_up)` 方法。

**`apps/main/src-tauri/src/engine/interception.rs`**

- 新增 `mouse_device: Option<InterceptionDevice>`（用 `INTERCEPTION_MAX_KEYBOARD + 1 ..= INTERCEPTION_MAX_DEVICE` 扫描 `interception_is_mouse`）。
- 新增 `send_mouse(button, is_up)`：构造 `InterceptionMouseStroke { state, flags: 0, rolling: 0, x: 0, y: 0, information: SIM_MARKER }`：
  - 5 个按钮各对应 `INTERCEPTION_MOUSE_{LEFT,RIGHT,MIDDLE,BUTTON_4,BUTTON_5}_BUTTON_{DOWN,UP}` 状态位
- 鼠标设备未找到时 `send_mouse` 直接返回（不降级；与键盘设备未找到时的行为对称）。

### 3. 物理按键 Hook 扩鼠标

**`apps/main/src-tauri/src/engine/burst.rs`**

- 新增 `WH_MOUSE_LL` + `MSLLHOOKSTRUCT` 钩子，与现有键盘钩子并存于同一消息循环线程。
- `mouse_hook_proc`：把 `wparam` 映射到 `(MouseButton, is_up)`：
  - `WM_LBUTTONDOWN/UP`、`WM_RBUTTONDOWN/UP`、`WM_MBUTTONDOWN/UP`、`WM_XBUTTONDOWN/UP`
  - X 键：从 `MSLLHOOKSTRUCT.mouseData >> 16` 取 `XBUTTON1/2` 判定
- `MSLLHOOKSTRUCT.dwExtraInfo == SIM_MARKER` 时跳过；否则尝试 `try_consume_injection(KeyId::Mouse(btn), is_up)`，命中跳过；剩余事件派发到 `engine.on_key_press(KeyId::Mouse(...))` / `on_key_release(...)`。
- `BurstEngine::on_key_press(KeyId)` / `on_key_release(KeyId)` 全部接 `KeyId`，规则匹配改为 `==` 比较 `KeyId`。
- `BurstRule.target_key` 改为 `KeyId` 后，`active_loops` 第三元从 `u32` 改为 `KeyId`，panic 兜底的 `key_up` 调用同步。
- 鼠标键的 OS-repeat：鼠标按住通常不会产生重复 LL 事件（与键盘 `LLKHF_REPEAT` 不同），无需特殊处理；但要忽略 `WM_MOUSEMOVE` / `WM_MOUSEWHEEL`（只处理按键事件）。

### 4. 前端 UI

**`apps/main/src/windows/panel/components/KeyCapture.tsx`**

- 键盘白名单 `BROWSER_VK` 扩展到约 120 项（在原表基础上补全）：
  - NumPad：`Numpad0-9` → 0x60-0x69、`NumpadAdd/Subtract/Multiply/Divide/Decimal/Enter` → 0x6B/0x6D/0x6A/0x6F/0x6E/0x0D
  - F13-F24 → 0x7C-0x87
  - OEM 标点：`Semicolon` 0xBA、`Equal` 0xBB、`Comma` 0xBC、`Minus` 0xBD、`Period` 0xBE、`Slash` 0xBF、`Backquote` 0xC0、`BracketLeft` 0xDB、`Backslash` 0xDC、`BracketRight` 0xDD、`Quote` 0xDE
  - 系统编辑键：Insert 0x2D、Delete 0x2E、Home 0x24、End 0x23、PageUp 0x21、PageDown 0x22
  - 修饰键独立位：CapsLock 0x14、ContextMenu 0x5D
  - NumLock 0x90、ScrollLock 0x91、Pause 0x13
- `KEY_NAMES` 同步扩。
- 鼠标录入：组件根 `<button>` 上加 `onMouseDown`（capturing 时阻止默认 + `e.button` 0/1/2/3/4 → Left/Middle/Right/X1/X2）。鼠标按下不会触发 `onBlur`，但要调 `e.preventDefault()` 防 button 抢焦点。
- 组件外部接口由 `value: number` / `onChange(vk)` 改为 `value: KeyId` / `onChange(KeyId)`。
- `keyLabel(KeyId)`：键盘走原表，鼠标返回「鼠标左键 / 右键 / 中键 / 侧键 1 / 侧键 2」。

**`apps/main/src/windows/panel/PanelApp.tsx`**

- `BurstRule` TS 接口字段类型改为 `KeyId`（与 Rust 共享 wire format）。
- 默认 `newRule()` 的 `trigger_key/target_key` 改为 `{ kind: 'keyboard', code: 0x51 }`（Q）/ `0x46`（F）。
- 4 个 `KeyCapture` 调用点接口签名同步。

| 场景 | SendInput | Interception | DD-HID |
|---|---|---|---|
| 键盘 trigger → 键盘 target（Hold + Toggle） | ✓ | ✓ | ✓ |
| 键盘 trigger → 鼠标 Left target | ✓ | ✓ | ✓ |
| 鼠标 Right trigger → 键盘 target | ✓ | ✓ | ✓ |
| 鼠标 X1 trigger → 鼠标 Left target | ✓ | ✓ | ✓（target X1/X2 应被 schema 拦下，trigger 允许） |

| 切换输入模式时鼠标 hook 不残留 | ✓ | ✓ | ✓ |

### 自循环回归

鼠标连发的 `target = trigger` 场景特别测：鼠标 Left 既是触发又是 target 时，`PENDING_INJECTIONS` 必须正确过滤自身注入，不形成无限循环。

## 已知约束 / 取舍

- **DD-HID 不支持 X1/X2 作为 target**：DD SDK 文档 `DD_btn` 值域只到 32。schema validate 阶段就拦下，UI 提示用户改用 SendInput / Interception 模式。X1/X2 作为 trigger（hook 端识别）则不受限。
- **WebView2 不会拦截鼠标事件**：与多媒体键不同，鼠标 onMouseDown 在前端能稳定捕获，无需后端临时挂 hook。
- **NumpadEnter / NumpadDecimal**：NumpadEnter 与 Enter 共用 VK 0x0D（靠扩展位区分），本期不区分；NumpadDecimal 用 `e.code === 'NumpadDecimal'` 显式映射 VK_DECIMAL 0x6E。
- **多媒体 / 系统键**（音量、播放、邮件等）：不在本期覆盖范围。
- **License gating 暂不做**：等兑换码体系完善后单独 PR，预留的 `MOUSE_BURST` 位本期保持 0 引用。
